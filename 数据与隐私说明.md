# 数据与隐私说明

## 1. 收集的数据

系统会收集以下数据：

- 姓名。
- 公司名称。
- 职务。
- 邮箱。
- 手机号。
- 公司画像。
- 问卷答案。
- 财务和运营指标。
- 诊断结果。
- 报告 token。
- 隐私同意版本。
- 提交时间。

## 2. 使用目的

数据用于：

- 生成企业经营健康诊断报告。
- 保存用户的完整报告。
- 通知 CXODEX 有新线索。
- 后续咨询沟通。
- 分析常见经营问题。

## 3. 不应做的事情

运营和维护人员不应：

- 对外公开单个企业的诊断数据。
- 在未获授权的情况下转发报告。
- 把用户手机号或邮箱导出到无关系统。
- 使用未确认的行业数据给用户下结论。

## 4. 用户删除请求

如果用户要求删除数据，应在 Supabase 中按以下字段查询：

- 公司名称。
- 邮箱。
- 手机号。
- report token。

确认后删除对应记录。

## 5. 生产环境安全建议

- Supabase 使用强密码。
- 不在代码中提交 `.env.local`。
- Resend API Key 只放在部署平台环境变量中。
- 定期检查 Vercel 环境变量权限。
- 不把完整数据库导出文件发送到个人微信或非受控网盘。
- 如果未来加入后台管理，必须增加管理员登录和权限控制。

## 6. 页面隐私提示文案

当前页面使用的隐私同意含义：

```text
我同意 CXODEX 保存本次企业诊断信息，用于报告生成和后续咨询沟通。
```

如果正式上线，建议在官网增加独立隐私政策页面，并在诊断页面链接到该页面。
