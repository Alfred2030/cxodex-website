# 部署上线指南

## 1. 推荐部署方式

推荐使用 Vercel 部署 Next.js 应用。

原因：

- 对 Next.js 支持最好。
- 自动构建和发布。
- 支持环境变量。
- 支持自定义域名。
- 后续可方便接入 `www.cxodex.com`。

## 2. 上线前准备

需要准备：

- GitHub 仓库。
- Vercel 账号。
- Supabase 项目。
- Resend 账号。
- CXODEX.com 域名 DNS 管理权限。

## 3. 创建 Supabase 数据库

1. 登录 Supabase。
2. 创建新项目。
3. 打开 SQL Editor。
4. 执行 `supabase/schema.sql`。
5. 获取 Postgres 连接字符串。

连接字符串填入部署环境变量：

```bash
DATABASE_URL="postgresql://postgres:password@db.project.supabase.co:5432/postgres"
```

## 4. 配置 Resend

1. 登录 Resend。
2. 验证发信域名，例如 `cxodex.com`。
3. 创建 API Key。
4. 配置发信邮箱。

环境变量：

```bash
RESEND_API_KEY="re_..."
REPORT_FROM_EMAIL="CXODEX <diagnosis@cxodex.com>"
LEAD_NOTIFY_EMAIL="hello@cxodex.com"
```

## 5. 部署到 Vercel

1. 将项目上传到 GitHub。
2. 打开 Vercel。
3. Import Git Repository。
4. Framework 选择 Next.js。
5. Build Command 使用默认值或：

```bash
npm run build
```

6. 添加环境变量。
7. 点击 Deploy。

## 6. 域名接入方案

### 方案 A：部署为子域名

推荐先使用：

```text
https://diagnosis.cxodex.com
```

DNS 设置：

```text
Type: CNAME
Name: diagnosis
Value: cname.vercel-dns.com
```

优点：

- 不影响现有官网。
- 上线风险低。
- 可以马上在官网添加“免费企业诊断”按钮。

### 方案 B：部署到 `www.cxodex.com/diagnosis`

如果现有官网也迁移到 Vercel，则可直接在同一项目中使用：

```text
https://www.cxodex.com/diagnosis
```

如果现有官网不在 Vercel，但仍想使用 `/diagnosis` 路径，需要当前官网平台支持 reverse proxy 或 rewrite，把 `/diagnosis` 转发到 Vercel 应用。

注意：DNS 不能直接把某个路径 `/diagnosis` 指向另一个服务。DNS 只能控制域名或子域名。

## 7. 上线验收清单

上线后逐项检查：

- 打开 `/diagnosis` 页面成功。
- 可以填写问卷。
- 摘要预览正常显示。
- 不填写姓名/公司时，提交被拒绝。
- 不勾选隐私同意时，提交被拒绝。
- 合法提交后生成完整报告。
- 完整报告链接可打开。
- PDF 下载可用。
- Supabase 中出现新记录。
- `hello@cxodex.com` 收到线索邮件。
- 移动端页面没有明显文字溢出。

## 8. 回滚方案

如果上线后发现问题：

1. 在 Vercel 中回滚到上一版本部署。
2. 或临时从官网移除诊断入口。
3. 保留 Supabase 数据，不要直接删除生产表。
