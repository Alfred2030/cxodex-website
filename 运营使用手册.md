# 运营使用手册

## 1. 日常使用流程

1. 用户访问诊断页面。
2. 用户填写问卷并查看摘要。
3. 用户提交姓名、公司、邮箱或手机号。
4. 系统生成完整报告。
5. CXODEX 收到线索邮件。
6. 运营人员根据报告中的 Top 风险进行跟进。

## 2. 如何跟进线索

建议优先联系以下用户：

- 综合健康分低于 55 分。
- Top 风险包含现金流、战略不清晰、KPI 未落地、产能利用率低。
- 公司规模、行业和 CXODEX 服务方向匹配。
- 用户填写了手机号。
- 用户打开完整报告后主动邮件或预约沟通。

## 4. 报告解释原则

对用户解释报告时，应遵守：

- 诊断结果是初步体检，不是最终咨询结论。
- 不夸大风险。
- 不使用未确认的行业基准。
- 对“待复核”项目，需要补充行业数据、财务数据或现场访谈。
- 优先讨论现金流、增长、战略、KPI、交付、质量和组织继任问题。

## 6. 常见问题

### 用户说没有收到邮件

当前系统默认邮件只通知 CXODEX，不自动给用户发送报告邮件。用户可直接在页面打开完整报告链接并保存 PDF。

### 用户没有填写手机号

只要填写了有效邮箱，也允许生成报告。

### 用户误填信息

第一版没有后台编辑功能。可让用户重新填写一次，生成新的报告。

### 用户希望删除数据

运营人员应根据用户请求，在 Supabase 中按公司、邮箱、手机号或 report token 查询并删除对应记录。
