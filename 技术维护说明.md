# 技术维护说明

## 1. 技术栈

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Zod
- Supabase Postgres
- Resend
- Node.js 内置 test runner

## 3. 修改问题库

问题定义在：

```text
src/lib/diagnosis/questions.ts
```

新增问题时需要设置：

- `id`
- `dimension`
- `kind`
- `title`
- `help`
- `conclusion`
- `threshold`

如果新增问题属于某个步骤，还要更新：

```ts
diagnosisSteps
```

## 4. 修改评分逻辑

评分逻辑在：

```text
src/lib/diagnosis/engine.ts
```

主要函数：

- `evaluateDiagnosis`
- `evaluateQuestion`
- `evaluateMetrics`
- `buildDimensionScore`

改动评分逻辑后必须更新或新增测试：

```text
tests/diagnosis.test.ts
```

## 5. 修改 API 校验

API 输入校验在：

```text
src/lib/diagnosis/schema.ts
```

当前校验包括：

- 答案对象。
- 财务/运营指标对象。
- 公司画像。
- lead 信息。
- 邮箱或手机号至少填一个。
- 隐私同意必须为 true。

## 6. 数据存储

生产环境：

```text
Supabase Postgres
```

本地开发如果没有 `DATABASE_URL`：

```text
.local/diagnosis-submissions.jsonl
```

生产环境必须配置 `DATABASE_URL`，不要依赖本地文件 fallback。

## 7. 邮件通知

邮件通知逻辑在：

```text
src/lib/server/email.ts
```

如果没有配置 `RESEND_API_KEY`，系统会跳过发邮件，但不会阻止报告生成。

## 8. 测试命令

```bash
npm test
npm run build
npm audit --audit-level=moderate
```

## 9. 已知设计决策

- 不伪造行业基准。
- 没有账号系统。
- 没有后台管理台。
- PDF 使用浏览器打印生成。
- 留资后完整报告通过 token 访问。
- 频率限制使用内存 Map，适合第一版；如果流量增大，应迁移到 Redis 或 Vercel KV。
